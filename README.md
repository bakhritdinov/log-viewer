# LogViewer C++

Высокопроизводительное десктопное приложение для просмотра и анализа логов из **Grafana (Loki)**, переписанное на **C++ 20** и **Qt 6**.

## 🚀 Возможности

- **Мгновенный поиск**: Поддержка LogQL для фильтрации логов в реальном времени.
...
- **Синхронизация маппингов**: Автоматическое получение связок namespace -> app из Grafana.

## 📦 Установка

### Windows (Chocolatey)
```powershell
choco install log-viewer --version=2.0.0
```

### macOS (Homebrew)
```bash
brew install --cask bakhritdinov/tap/log-viewer
```

### Ubuntu (PPA)
```bash
sudo add-apt-repository ppa:bakhritdinov/log-viewer
sudo apt update
sudo apt install log-viewer
```

## 📦 Сборка проекта

### Требования
- CMake 3.16 или выше
- Qt 6 (модули: Core, Quick, Network, Svg)
- Компилятор с поддержкой C++ 20 (GCC 10+, Clang 11+, MSVC 2019+)

### Инструкция по сборке

```bash
# 1. Клонирование репозитория
git clone <repository-url>
cd log-viewer

# 2. Создание директории сборки
mkdir build && cd build

# 3. Конфигурация
cmake ..

# 4. Сборка
make -j$(nproc)
```

## 🖥 Использование

### Настройка подключения
1. Запустите приложение.
2. Нажмите на иконку шестеренки (⚙️) в правом верхнем углу.
3. Введите данные для подключения к вашему экземпляру Grafana:
   - **URL**: Базовый URL Grafana.
   - **Loki UID**: UID источника данных Loki.
   - **Credentials**: Имя пользователя и пароль (Basic Auth) или Token.
   - **NS/App Labels**: Имена меток, по которым группируются логи (по умолчанию `_namespace` и `_appName`).
4. Нажмите **Save**.

## 📊 Требования к данным (Loki Labels)

Для корректной работы фильтрации и отображения, ваши логи в Loki должны иметь определенную структуру меток (labels).

### Обязательные метки (настраиваются в приложении)
Приложение использует две основные метки для построения дерева навигации:
- **Namespace Label**: Метка для группировки по окружению/пространству имен (по умолчанию `_namespace`).
- **App Label**: Метка для идентификации конкретного приложения/сервиса (по умолчанию `_appName`).

### Рекомендуемые метки (автоматическое распознавание)
Приложение умеет автоматически извлекать и подсвечивать следующие поля из метаданных (labels):
- `level`: Используется для цветовой индикации (ERROR, WARN, INFO). Если метка отсутствует, приложение попытается найти уровень в тексте сообщения.
- `trace_id` или `traceId`: Используется для быстрой фильтрации всех логов в рамках одного запроса (клик по ID в таблице).
- `pod` или `host.name`: Отображается как источник лога.

### Структура сообщения
Приложение поддерживает стандартные форматы вывода Grafana:
- **Time/ts**: Временная метка сообщения.
- **Line/message**: Основной текст сообщения.
